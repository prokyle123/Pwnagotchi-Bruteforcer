A small appliance, not a pile of terminal windows

Make authorized capture review organized, measurable, and easier to explain.

BruteForcer watches a configured capture directory, handles one queued job at a time, keeps a persistent record of what happened, and brings operational visibility into one local web interface.

Capture triageGrade, group duplicates, and see why a job was queued or deferred.
Measured performanceTrack tested-key counters and elapsed time instead of relying only on terminal output.
Device awarenessKeep CPU temperature, RAM, swap, load, and fan telemetry visible beside the workload.
Core capabilities

Designed for visibility at every stage of an authorized workflow.

▣

Command Center

See the active job, queue, retry state, activity timeline, current throughput, recent averages, and completed-job performance from one page.

⌁

Capture Intelligence

Use quality grades, plain-English queue reasons, duplicate grouping, review filters, per-capture history, and separate reports pages.

◌

System health

Surface CPU temperature, RAM, swap, load average, and optional PWM/RPM fan telemetry alongside the active workload.

≋

Mutator Lab

Expose strategy and workload settings with candidate caps, candidate-family breakdowns, build history, and estimated pass duration.

↻

Recovery-aware state

Keep persistent progress, retries, job history, and an active-job journal so interruptions are easier to understand after a crash or reboot.

⌗

Reports and exports

Review per-SSID history, wordlist effectiveness, and exportable summaries for authorized testing records and lab documentation.

Workflow

From capture triage to an explainable job history.

01Capture intakeNew authorized capture files appear in the configured directory.
02TriageQuality grades and duplicate grouping help prioritize what to review.
03Queue controlPause, resume, defer, requeue, skip, or mark review outcomes locally.
04Measured runTrack progress, performance, thermals, and system health as the job proceeds.
05History + reportsKeep clear outcomes, retry state, and reporting context after the work ends.

Local, focused, explainable

No cloud account. No mystery status. No hidden workload settings.

The local interface runs on the Pwnagotchi and is intended for lab, recovery, and security-assessment environments where you own the networks and captures—or have explicit written authorization to assess them.

Read install and upgrade notes

What this project focuses on

  • Authorized capture-file triage and local queue management.
  • Transparent device performance and thermal visibility.
  • Persistent, explainable job state and reports.
  • Safer lab operations through clear controls and audit context.
  • Redacted public documentation and responsible sharing.
Authorized use only: Use BruteForcer only with wireless networks, capture files, and credentials you own or have explicit permission to assess. This project is for recovery testing, security assessment, and controlled lab work—not unauthorized access.
